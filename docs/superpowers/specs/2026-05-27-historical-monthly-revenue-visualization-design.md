# 历史月度营收导入与客户详情可视化设计

## 背景

Fishpool 当前已有客户营收模型 `CustomerRevenue`，按客户和月份记录营收：

```text
CustomerRevenue(customer, month, revenue)
```

现有营收导入能力已经支持 `.xlsx` 和 `.csv`，并要求数据包含：

```text
month, customer name, revenue
```

其中 `month` 已支持 `YYYY-MM`、`YYYY-MM-DD` 等格式。这个模型天然支持导入前几年销售数据，因为历史销售数据只需要按月写入对应月份即可。

## 明确口径

1. 历史销售数据全部按月度导入。
2. 不支持、也不需要导入年度总销售额。
3. 年度营收由系统根据月度营收自动汇总。
4. 同一客户同一月份重复导入时，继续沿用现有逻辑：更新该月份营收，而不是新增重复记录。
5. 客户匹配仍使用客户名称完全匹配，避免历史销售数据写入错误客户。

## 导入数据格式

历史销售数据继续使用现有营收导入格式：

```csv
month,customer name,revenue
2021-01,ABC客户,1200
2021-02,ABC客户,1500
2022-01,ABC客户,1800
2023-12,ABC客户,2300
```

推荐规则：

1. 每一行代表一个客户某一个月份的销售额。
2. `month` 使用 `YYYY-MM`，例如 `2021-01`。
3. `revenue` 使用数字，不带货币符号。
4. 如果只有年度总额，用户需要先在外部拆分为月度数据后再导入；系统不做年度总额拆分。

## 后端设计

### 目标

新增客户维度的营收汇总接口，让客户详情页可以直接获取：

1. 历年营收汇总。
2. 月度营收明细。
3. 可选年份筛选后的月度明细。

### 建议接口

```http
GET /api/customers/{id}/revenue-summary/
```

可选查询参数：

```text
year=2023
```

### 返回结构

```json
{
  "yearly": [
    {"year": 2021, "revenue": "16800.00"},
    {"year": 2022, "revenue": "23600.00"},
    {"year": 2023, "revenue": "30100.00"}
  ],
  "monthly": [
    {"month": "2023-01", "revenue": "1200.00"},
    {"month": "2023-02", "revenue": "1500.00"}
  ]
}
```

### 聚合逻辑

年度营收使用数据库聚合计算：

1. 按 `customer_id` 过滤。
2. 从 `CustomerRevenue.month` 提取年份。
3. 对同一年内所有月度 `revenue` 求和。
4. 按年份升序返回。

月度明细：

1. 按 `month` 升序返回。
2. 如果传入 `year`，只返回该年的月度数据。
3. 如果不传 `year`，返回该客户全部月度数据，或后续按性能需要限制为最近若干年。

## 前端设计

### 客户详情页位置

在 `CustomerDetailPage.vue` 中增加“历年营收”模块。推荐放在联系人列表卡片上方或联系人列表卡片内部上方，因为用户提出希望在联系人列表上方看到历年营收柱状图。

建议结构：

```text
右侧区域
├── 三维评分
├── 联系人列表
│   ├── 历年营收柱状图
│   └── 联系人表格
```

### 图表

复用项目已有的 ECharts。

柱状图配置：

1. X 轴：年份。
2. Y 轴：年度营收。
3. Tooltip：显示年份和营收金额。
4. 点击某一年柱子时，可筛选下方月度明细表。

### 数据表

在柱状图下方增加月度明细表：

| 月份 | 营收 |
|---|---:|
| 2023-01 | 1200.00 |
| 2023-02 | 1500.00 |

可选交互：

1. 年份下拉框：查看全部或指定年份。
2. 点击柱状图年份：自动切换到该年份。
3. “查看全部”按钮：恢复全部月度明细。

## 与现有功能的关系

1. 客户列表中的“去年营收”和“上季度营收”继续保留。
2. 综合筛选中的“去年营收”和“上季度营收”继续基于月度数据聚合。
3. 历年营收柱状图不影响现有客户、联系人、Weekly Report 逻辑。
4. 营收导入接口不需要新增年度总额模式。

## 测试建议

### 后端

新增客户营收汇总接口测试：

1. 同一客户多个月份跨年份数据能正确按年汇总。
2. 不同客户的营收不会互相混入。
3. `year=2023` 时只返回 2023 年月度明细。
4. 无营收客户返回空数组。

### 前端

1. 客户详情页能请求并渲染历年营收柱状图。
2. 没有营收数据时显示空状态。
3. 点击年份或选择年份后，月度明细表正确变化。
4. 切换客户详情页时图表销毁并重新初始化，避免 ECharts 实例泄漏。

## 实施顺序

1. 后端增加 `revenue-summary` action 和测试。
2. 前端增加 `getCustomerRevenueSummary` API 封装。
3. 客户详情页加载汇总数据。
4. 增加历年营收柱状图。
5. 增加月度明细表和年份筛选。
6. 运行后端测试和前端构建验证。
