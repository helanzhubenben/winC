# 客户管理系统需求文档

## 一、系统概述

基于Excel工作表"使用说明"和"General-Fishpool"构建的客户关系管理系统。

**核心功能**：客户信息管理和客户分级评估

---

## 二、数据来源分析

### 工作表1：使用说明

定义了客户开发流程的关键状态：

1. **new opportunity** - 新商机：开发过程中了解到的客户需求
2. **New Inquire** - 新询价：由商机发展来的询价
3. **PO** - 订单：由询价发展来的订单
4. **follow** - 跟进：一般性跟进工作
5. **Customer Feedback** - 客户反馈：需要发给运营部、质量部改善
6. **Close** - 关闭：已取消的需求

**业务规则**：
- 每月第一个周一根据weekly Report筛选new opportunity
- 更新pipeline状态
- 填写database信息
- 利用dashboard管理任务

### 工作表2：General-Fishpool (客户池)

**规模**：341条客户记录 × 16列

**数据结构**：

#### 1. 客户基本信息
- Nr. (编号)
- Update (更新日期)
- Client (客户名称)
- Business Model (业务模式)
  - Farming：维护型客户
  - Hunting：开发型客户
- Area (区域)：East Region, South Region等
- Address (地址)

#### 2. 客户潜力评估 (三维评分体系)

**X轴 - 客户潜力**
- Description：客户业务描述、产品类型、任务分布
- Score X：0-100分

**Y轴 - 竞争环境**
- Description：竞争对手分析、市场压力
- Score Y：0-100分

**Z轴 - 关键人关系**
- Key Person：关键联系人姓名、电话
- Score Z：0-100分

#### 3. 客户分级系统

基于X、Y、Z三个维度的评分组合：

- **Level A**：X↑, Y↑, Z↑ (三项都高)
- **Level B**：X↑, Y↑ or Z↑ (至少两项高)
- **Level C**：X↑, Y↓, Z↓ (客户潜力高，但竞争或关系弱)
- **Level D**：X↓, Y↓, Z↓ (三项都低)

#### 4. 战略信息
- Client Strategy：客户策略描述
- Potential Contribution (KEUR/year)：年度潜在贡献(千欧元)
- Remark：备注信息

## 三、系统功能设计

### 3.1 客户信息管理

**功能列表**：
- 客户列表展示（支持分页、搜索、筛选）
- 客户详情查看
- 新增客户
- 编辑客户信息
- 删除客户（软删除）

**筛选条件**：
- 业务模式（Farming/Hunting）
- 区域
- 客户等级（A/B/C/D）
- 评分范围（X/Y/Z）

### 3.2 客户分级评估

**评分管理**：
- X轴评分（客户潜力）：0-100
- Y轴评分（竞争环境）：0-100
- Z轴评分（关键人关系）：0-100
- 自动计算客户等级（A/B/C/D）

**评分规则**：
```
Level A: X >= 70 AND Y >= 70 AND Z >= 70
Level B: (X >= 70 AND Y >= 70) OR (X >= 70 AND Z >= 70) OR (Y >= 70 AND Z >= 70)
Level C: X >= 70 AND (Y < 70 OR Z < 70)
Level D: 其他情况
```

### 3.3 关键人管理

**功能**：
- 每个客户可添加多个联系人
- 标记关键联系人
- 记录联系人信息：姓名、职位、电话、邮箱
- 关系评分

### 3.4 数据统计

**统计维度**：
- 按等级统计客户数量（A/B/C/D）
- 按区域统计客户分布
- 按业务模式统计（Farming/Hunting）
- 潜在收入汇总（按等级、按区域）

### 3.5 数据导入导出

**功能**：
- 从Excel导入客户数据
- 导出客户数据到Excel
- 支持批量导入

## 四、数据库设计

### 4.1 客户表 (customers)

```sql
CREATE TABLE customers (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    client_name VARCHAR(200) NOT NULL COMMENT '客户名称',
    business_model VARCHAR(20) COMMENT '业务模式: Farming/Hunting',
    area VARCHAR(100) COMMENT '区域',
    address TEXT COMMENT '地址',
    
    -- 客户潜力 (X轴)
    description_x TEXT COMMENT '客户描述',
    score_x INT DEFAULT 0 COMMENT '客户潜力评分 0-100',
    
    -- 竞争环境 (Y轴)
    description_y TEXT COMMENT '竞争环境描述',
    score_y INT DEFAULT 0 COMMENT '竞争评分 0-100',
    
    -- 关键人关系 (Z轴)
    score_z INT DEFAULT 0 COMMENT '关系评分 0-100',
    
    -- 分级和战略
    level VARCHAR(1) COMMENT '客户等级: A/B/C/D',
    client_strategy TEXT COMMENT '客户策略',
    potential_contribution DECIMAL(10,2) COMMENT '年度潜在贡献(KEUR)',
    
    remark TEXT COMMENT '备注',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态: active/inactive',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_level (level),
    INDEX idx_area (area),
    INDEX idx_business_model (business_model),
    INDEX idx_status (status)
) COMMENT='客户信息表';
```

### 4.2 联系人表 (contacts)

```sql
CREATE TABLE contacts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL COMMENT '客户ID',
    name VARCHAR(100) NOT NULL COMMENT '姓名',
    position VARCHAR(100) COMMENT '职位',
    phone VARCHAR(50) COMMENT '电话',
    email VARCHAR(100) COMMENT '邮箱',
    is_key_person BOOLEAN DEFAULT FALSE COMMENT '是否关键人',
    relationship_score INT DEFAULT 0 COMMENT '关系评分',
    remark TEXT COMMENT '备注',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
    INDEX idx_customer (customer_id),
    INDEX idx_key_person (is_key_person)
) COMMENT='联系人表';
```

### 4.3 客户历史记录表 (customer_history)

```sql
CREATE TABLE customer_history (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL COMMENT '客户ID',
    field_name VARCHAR(50) COMMENT '字段名',
    old_value TEXT COMMENT '旧值',
    new_value TEXT COMMENT '新值',
    changed_by VARCHAR(100) COMMENT '修改人',
    changed_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '修改时间',
    
    FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
    INDEX idx_customer (customer_id),
    INDEX idx_changed_at (changed_at)
) COMMENT='客户变更历史表';
```

## 五、技术架构建议

**选项A：Spring Boot (Java)**

- Spring Data JPA
- MySQL 8.0
