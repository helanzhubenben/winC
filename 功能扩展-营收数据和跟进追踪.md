# 功能扩展实现计划：营收数据管理 + 跟进追踪

## 一、概述

为现有客户管理系统添加两个核心功能模块：
1. **营收数据管理**：记录和分析客户的实际营收数据，支持历史数据导入和趋势分析
2. **跟进追踪系统**：完整的客户跟进生命周期管理，包括跟进计划、执行记录和状态追踪

这两个功能将与现有的客户分级系统深度集成，形成"潜在评估 → 实际营收 → 持续跟进"的完整闭环。

## 二、数据库设计扩展

### 2.1 新增表：revenue_records (营收记录表)

```sql
CREATE TABLE revenue_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL COMMENT '客户ID',
    
    -- 营收基本信息
    record_date DATE NOT NULL COMMENT '记录日期',
    record_type VARCHAR(20) NOT NULL COMMENT '记录类型: actual/forecast/target',
    period_type VARCHAR(20) NOT NULL COMMENT '周期类型: monthly/quarterly/yearly',
    period_value VARCHAR(20) NOT NULL COMMENT '周期值: 2026-01, 2026-Q1, 2026',
    
    -- 营收金额
    revenue_amount DECIMAL(12,2) NOT NULL COMMENT '营收金额(EUR)',
    currency VARCHAR(10) DEFAULT 'EUR' COMMENT '货币单位',
    
    -- 业务分类
    business_type VARCHAR(50) COMMENT '业务类型: Sorting/Rework/Inspection等',
    project_name VARCHAR(200) COMMENT '项目名称',
    
    -- 附加信息
    remark TEXT COMMENT '备注',
    data_source VARCHAR(50) COMMENT '数据来源: manual/import/system',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    created_by VARCHAR(100) COMMENT '创建人',
    
    FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
    INDEX idx_customer (customer_id),
    INDEX idx_date (record_date),
    INDEX idx_period (period_type, period_value),
    INDEX idx_type (record_type)
) COMMENT='客户营收记录表';
```

### 2.2 新增表：follow_up_plans (跟进计划表)

```sql
CREATE TABLE follow_up_plans (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL COMMENT '客户ID',
    
    -- 计划信息
    plan_title VARCHAR(200) NOT NULL COMMENT '计划标题',
    plan_type VARCHAR(50) NOT NULL COMMENT '计划类型: visit/call/email/meeting',
    priority VARCHAR(20) DEFAULT 'medium' COMMENT '优先级: high/medium/low',

    planned_date DATE NOT NULL COMMENT '计划日期',
    planned_time VARCHAR(20) COMMENT '计划时间: AM/PM/具体时间',
    estimated_duration INT COMMENT '预计时长(分钟)',
    
    -- 计划内容
    objective TEXT COMMENT '跟进目标',
    content TEXT COMMENT '计划内容',
    
    -- 状态管理
    status VARCHAR(20) DEFAULT 'planned' COMMENT '状态: planned/in_progress/completed/cancelled',
    
    -- 责任人
    responsible_person VARCHAR(100) COMMENT '负责人',
    participants TEXT COMMENT '参与人员(JSON数组)',
    
    -- 关联信息
    related_opportunity_id BIGINT COMMENT '关联商机ID',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    created_by VARCHAR(100) COMMENT '创建人',
    
    FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
    INDEX idx_customer (customer_id),
    INDEX idx_date (planned_date),
    INDEX idx_status (status),
    INDEX idx_priority (priority)

```

### 2.3 新增表：follow_up_records (跟进记录表)

```sql
CREATE TABLE follow_up_records (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL COMMENT '客户ID',
    plan_id BIGINT COMMENT '关联计划ID(可为空)',
    
    -- 跟进基本信息
    follow_up_date DATE NOT NULL COMMENT '跟进日期',
    follow_up_type VARCHAR(50) NOT NULL COMMENT '跟进方式: visit/call/email/meeting',
    
    -- 跟进内容
    title VARCHAR(200) NOT NULL COMMENT '标题',
    content TEXT NOT NULL COMMENT '跟进内容/会议纪要',
    outcome TEXT COMMENT '跟进结果',
    
    -- 客户反馈
    customer_feedback TEXT COMMENT '客户反馈',
    customer_satisfaction VARCHAR(20) COMMENT '客户满意度: very_satisfied/satisfied/neutral/dissatisfied',
    
    -- 商机信息
    opportunity_identified BOOLEAN DEFAULT FALSE COMMENT '是否发现商机',
    opportunity_description TEXT COMMENT '商机描述',
    estimated_value DECIMAL(12,2) COMMENT '预估价值(EUR)',
    
    -- 下一步行动
    next_action TEXT COMMENT '下一步行动',
    next_follow_up_date DATE COMMENT '下次跟进日期',
    
    -- 附件和关联
    attachments TEXT COMMENT '附件列表(JSON数组)',
    related_contacts TEXT COMMENT '相关联系人(JSON数组)',
    
    -- 执行人
    executor VARCHAR(100) COMMENT '执行人',
    participants TEXT COMMENT '参与人员(JSON数组)',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    created_by VARCHAR(100) COMMENT '创建人',
    
    FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
    FOREIGN KEY (plan_id) REFERENCES follow_up_plans(id) ON DELETE SET NULL,
    INDEX idx_customer (customer_id),
    INDEX idx_date (follow_up_date),
    INDEX idx_plan (plan_id),
    INDEX idx_opportunity (opportunity_identified)
) COMMENT='客户跟进记录表';
```

### 2.4 新增表：customer_follow_up_status (客户跟进状态表)

```sql
CREATE TABLE customer_follow_up_status (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    customer_id BIGINT NOT NULL UNIQUE COMMENT '客户ID',
    
    -- 跟进状态
    current_stage VARCHAR(50) NOT NULL COMMENT '当前阶段: new_opportunity/new_inquire/po/follow/customer_feedback/close',
    stage_updated_at DATETIME COMMENT '阶段更新时间',
    
    -- 跟进统计
    total_follow_ups INT DEFAULT 0 COMMENT '总跟进次数',
    last_follow_up_date DATE COMMENT '最后跟进日期',
    next_follow_up_date DATE COMMENT '下次跟进日期',
    
    -- 跟进频率
    follow_up_frequency VARCHAR(20) COMMENT '跟进频率: daily/weekly/biweekly/monthly/quarterly',
    days_since_last_follow INT COMMENT '距上次跟进天数',
    
    -- 商机统计
    opportunities_count INT DEFAULT 0 COMMENT '商机数量',
    inquiries_count INT DEFAULT 0 COMMENT '询价数量',
    orders_count INT DEFAULT 0 COMMENT '订单数量',
    
    -- 状态标记
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否活跃',
    needs_attention BOOLEAN DEFAULT FALSE COMMENT '需要关注',
    attention_reason TEXT COMMENT '关注原因',
    
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (customer_id) REFERENCES customers(id) ON DELETE CASCADE,
    INDEX idx_stage (current_stage),
    INDEX idx_next_date (next_follow_up_date),
    INDEX idx_active (is_active),
    INDEX idx_attention (needs_attention)
) COMMENT='客户跟进状态汇总表';
```

## 三、核心功能设计

### 3.1 营收数据管理模块

#### 功能列表

1. **营收数据录入**
   - 手动录入单条营收记录
   - 批量导入Excel营收数据
   - 支持三种记录类型：实际营收(actual)、预测营收(forecast)、目标营收(target)
   - 支持多种周期：月度、季度、年度

2. **营收数据查询**
   - 按客户查询营收记录
   - 按时间范围筛选
   - 按记录类型筛选
   - 按业务类型筛选

3. **营收数据分析**
   - 客户营收趋势图（折线图）
   - 实际 vs 预测 vs 目标对比
   - 营收达成率计算
   - 按业务类型分布（饼图）
   - Top N客户营收排行

4. **营收数据导入**
   - 提供Excel导入模板
   - 支持批量导入历史数据
   - 数据验证和错误提示

5. **营收与评估关联**
   - 实际营收与潜在贡献对比
   - 营收异常预警

### 3.2 跟进追踪管理模块

#### 功能列表

1. **跟进计划管理**
   - 创建/编辑/删除跟进计划
   - 计划状态管理（计划中/进行中/已完成/已取消）
   - 计划提醒功能

2. **跟进记录管理**
   - 记录跟进活动
   - 记录客户反馈
   - 标记商机发现
   - 设置下次跟进时间

3. **跟进状态追踪**
   - 客户跟进阶段管理（new opportunity → new inquire → PO → follow → customer feedback → close）
   - 阶段流转记录
   - 活跃度监控

4. **跟进日历视图**
   - 月历/周历/日历视图
   - 拖拽调整计划时间
   - 颜色标记优先级

5. **跟进筛选功能**
   - 按跟进阶段筛选客户
   - 按最后跟进时间筛选
   - 按下次跟进时间筛选
   - 按是否需要关注筛选

## 五、技术要点

### 5.1 数据同步机制

**客户表字段自动更新**：
- 当创建营收记录时，自动更新customers表的actual_revenue_ytd、last_revenue_date
- 当创建跟进记录时，自动更新customers表的last_follow_up_date、follow_up_stage
- 使用数据库触发器或应用层事件监听实现

### 5.2 跟进阶段流转规则

```
new_opportunity (新商机)
    ↓
new_inquire (新询价)
    ↓
po (订单)
    ↓
follow (跟进)
    ↓
customer_feedback (客户反馈)
    ↓
close (关闭)
```

### 5.3 Excel导入模板格式

**营收数据导入模板**：
```
| 客户名称 | 记录日期 | 记录类型 | 周期类型 | 周期值 | 营收金额 | 业务类型 | 项目名称 | 备注 |
```

### 5.4 前端组件复用

- 使用Element Plus的日历组件
- 使用ECharts进行数据可视化
- 封装通用的筛选器组件
- 封装通用的表单组件
